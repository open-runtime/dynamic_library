# Quickstart: Dynamic Library Core

## 1. Overview
The `Dynamic Library Core` module provides robust cross-platform utilities for resolving, loading, and debugging dynamic libraries (`.so`, `.dylib`, `.dll`) in Dart and Flutter applications. It also provides high-level initialization functions for seamlessly integrating `flutter_rust_bridge` (FRB) generated libraries, automatically handling platform-specific naming conventions, paths, and dependency diagnostics.

## 2. Import
```dart
import 'package:dynamic_library/dynamic_library.dart';
```

## 3. Loading Libraries

Most applications will use the high-level `loadDynamicLibrary` or `initializeFrbLibrary` functions.

### Loading a Standard Dynamic Library
Provides verbose diagnostics via `LoadDynamicLibraryException` if the library or its downstream system dependencies are missing.
```dart
import 'package:dynamic_library/dynamic_library.dart';

try {
  // Automatically prepends 'lib' on Unix and appends the correct extension
  final library = loadDynamicLibrary(
    libraryName: 'my_native_lib',
    searchPath: '/usr/local/lib', // Optional: Specific directory to search
  );
} on LoadDynamicLibraryException catch (e) {
  print('Failed to load: $e');
}
```

### Loading an Exact Dynamic Library Filename
Useful for libraries with non-standard naming schemes (e.g., versioned Linux shared objects like `libonnxruntime.so.1.23.2`), bypassing platform naming conventions.
```dart
import 'package:dynamic_library/dynamic_library.dart';

final rawLibrary = loadDynamicLibraryRaw(
  fileName: 'libonnxruntime.so.1.23.2',
  searchPath: '/opt/local/lib',
);
```

## 4. Flutter Rust Bridge (FRB) Utilities

These functions assist with loading libraries generated by `flutter_rust_bridge`.

### Initializing a Flutter Rust Bridge (FRB) Library
The `initializeFrbLibrary` helper handles local resolution, asset bytes resolution, and platform quirks (e.g., macOS FRB libraries omitting the `lib` prefix).
```dart
import 'dart:io';
import 'package:dynamic_library/dynamic_library.dart';
// Import your FRB generated code
import 'rust/generated/frb_generated.dart' show RustLib;

Future<void> initMyRustLib() async {
  await initializeFrbLibrary(
    packageName: 'my_rust_package',
    rustLibInit: RustLib.init,
    // Use 'true' during local `dart run` development, 
    // or 'false' if providing libPath/libBytes
    assumeLocalResolution: true, 
  );
}
```

### FRB Path and Name Resolution
These utilities account for FRB's convention where macOS libraries are NOT prefixed with `lib`, while Linux libraries ARE prefixed.
```dart
import 'package:dynamic_library/dynamic_library.dart';

// Returns 'my_rust_package.dylib' on macOS, 'libmy_rust_package.so' on Linux
final String frbName = frbDynamicLibraryName('my_rust_package');

// Resolves the full path applying the same FRB prefix-strip workaround for macOS
final String frbPath = frbFullLibraryPath('my_rust_package');
```

## 5. Path and Name Resolution API

You can easily query how dynamic libraries are named or where they are stored on the current OS:
```dart
import 'package:dynamic_library/dynamic_library.dart';

// Returns '.dll' on Windows, '.dylib' on macOS/iOS, '.so' on Linux/Android
final String extension = systemLibExtension();

// Returns 'lib' on Unix-based OSs, '' on Windows
final String prefix = libraryPrefix();

// Returns 'libmy_library.so' on Unix, 'my_library.dll' on Windows
final String fullName = fullLibraryName('my_library', includePrefix: true);

// Resolve full path using standard libraryPrefix rules
final String fullPath = fullLibraryPath('my_library', searchPath: '/opt/lib');

// Retrieve the default directory containing Flutter assets
final String assetsDir = flutterAssetsDirectory(localDebugMode: false);

// Retrieve the default location from which dynamic libraries are loaded
final String defaultLibDir = defaultLibraryDirectory(localDebugMode: false);
```

## 6. Environment and Diagnostics

### Environment Checking
You can verify if the current executable is `dart` (e.g., during `dart run` or testing) vs a compiled application.
```dart
import 'package:dynamic_library/dynamic_library.dart';

final bool isRunningInDart = isDart();
```

### Dependency Checking
The library internally calls OS-specific tools to trace why a dynamic library fails to load. You can also call it directly:
```dart
import 'dart:io';
import 'package:dynamic_library/dynamic_library.dart';

// Calls `dumpbin /DEPENDENTS` (Windows), `otool -L` (macOS), or `ldd` (Linux)
final ProcessResult result = callOSDependencyCheck('/path/to/my_library.so');
print(result.stdout);
```

## 7. Using the DLib Wrapper Class

If you want to store metadata alongside your loaded dynamic library (such as the resolved path or filename), you can use the `DLib` wrapper class:

```dart
import 'dart:ffi';
import 'package:dynamic_library/dynamic_library.dart';

// 1. Load the raw DynamicLibrary
final DynamicLibrary myLib = loadDynamicLibrary(libraryName: 'my_library');

// 2. Wrap it in DLib for easier path and metadata access
final DLib dlib = DLib('/path/to/libmy_library.so', myLib);

print('Library: ${dlib.library}');
print('Path: ${dlib.path}');
print('Loaded: ${dlib.fileName} with extension ${dlib.extension}');
```

## 8. Configuration Details

For `loadDynamicLibrary` and `loadDynamicLibraryRaw`:
- **`searchPath`**: Used to designate a target directory to search. *Note: `searchPath` is recommended for Dart servers or CLI applications, but not for bundled cross-platform Flutter applications.*

For `initializeFrbLibrary`:
- **`assumeLocalResolution`**: When `true` (default), it resolves the library relative to `Directory.current` under standard FRB output paths. Must be set to `false` if `libPath` or `libBytes` are explicitly provided.
- **`libPath`**: Allows loading the library from an explicit string path.
- **`libBytes` & `libBytesCacheDir`**: Supports byte-based resolution. For example, if extracting a dynamic library from a Flutter asset bundle, pass the `ByteData` and a writable `Directory` to cache the library file to disk. Both parameters must be set together.

## 9. Exceptions

### `LoadDynamicLibraryException`
A custom exception thrown when `loadDynamicLibrary` or `loadDynamicLibraryRaw` fails. It provides verbose details including:
- Expected path and search path
- Current working directory
- Missing system dependencies via OS-specific tools (ldd, otool, dumpbin)

## 10. Related Modules
- **`flutter_rust_bridge`**: Tightly integrated internally to open `ExternalLibrary` references seamlessly via `frb_init.dart`.
- **`dart:ffi`**: The standard Dart Foreign Function Interface; this module returns native `DynamicLibrary` objects from `dart:ffi`.
- **`dart:io`**: Used internally for resolving OS-level platform properties, directories, and running OS-specific dependency checks.
